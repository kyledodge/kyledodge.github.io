I have a love-hate relationship with blogs. As a developer, I use blogs daily when they turn up when searching how to use a feature or fix an error. They are invaluable, and I’m not sure how I would do my job without them. Though when it comes to actually writing my own blog, I tend to lose sight of this same value I could be giving back to others. To kick this off, I wanted to show how I created this blog which doesn’t cost me a dime, and how much fun the process itself was.

Jekyll

Jekyll is an open source static site generator used to transform files (like markdown) into a static html site. It was created back in 2008 by Tom Preston-Werner - the co-founder of Github. As such, it has some great integration with Github that I’ll get into next. My favorite part about Jekyll is the philosophy behind it and the following quote from it’s README file:

Jekyll does what you tell it to do — no more, no less. It doesn't try to outsmart users by making bold assumptions, nor does it burden them with needless complexity and configuration. Put simply, Jekyll gets out of your way and allows you to concentrate on what truly matters: your content.

Getting Started

Jekyll’s installation documentation is top-notch, so I won’t try and duplicate it here. Follow the steps for the environment you want to use and you should be up and running within a few minutes. Here are a few helpful tutorials I used as well:

  • https://www.smashingmagazine.com/2014/08/build-blog-jekyll-github-pages/
  • https://help.github.com/articles/setting-up-your-github-pages-site-locally-with-jekyll/

Themes

Once you are up and running, you’ll want to apply a theme to get rid of the default look and feel. There are many free themes to choose from at http://jekyllthemes.org. You can even clone the theme’s git repository and use that as a starting point to get up and running quickly. That is what I wound up doing and it make the configuration process much simpler.

Github Pages

Now that you've got your blog setup, you need a place to host it. Fortunately Github allows you to host a site for free and even includes continuous integration and deployment. As soon as you commit your files, they will be compiled and deployed to your pages site automatically.

To get started, head over to https://pages.github.com for instructions on setting up github pages to host this blog content. You are basically creating a git repository called username.github.io and adding your files to it. Github is configured to build and serve this project as soon as files are pushed to the repository, and will show up almost instantly on the site.

Custom Domain

You’ll likely want to setup a custom domain to use for your blog. I would recommend Cloudflare as a domain registrar as they have a wholesale rate and include whois privacy at no charge. Once you purchase your domain (or if you have one already), you'll need to configure your DNS servers to point to Cloudflare (more on this in the next step). Once the DNS servers are updated, save the changes and allow the changes to propogate. You can always check https://www.whatsmydns.net/ to see the propagation status.

Cloudflare

Cloudflare is a great service and with a free account you get access to powerful features like worldwide caching, SSL and DDoS protection. Register for a free Cloudflare account and navigate to the “Add a Site” area. Once you add your site, head over to the “DNS” section to wrap up the Github pages configuration. For this example, I’m going to show www.mydomain.com using a CNAME record. Input “www” as the name and your github pages url as the domain name. Hit the add record button and Cloudflare will now respond to www.domain.com with your github pages site. Navigate to the “Crypto” option and enable SSL with the flexible option. This will allow your users to visit your site under https with a Cloudflare certificate. The great part is you don't have to manage this certificate, and it's free. One more thing you will need to do is head back to your Github account and under the repository settings, set the custom domain field to www.mydomain.com.

Disqus

People are going to want to comment on your blog, so you’ll want a place for them to do that.
